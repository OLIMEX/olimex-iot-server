Olimex IoT Glossary

Back to OlimexIoT Help

Action

Action - A plugin function which performs certain task.

For example action can send an e-mail to the user or a command to IoT Device.

Back

Filter

Filter - A plugin function which is applied over the values of property to reduce or enhance certain of property's characteristics.

Back

Device

IoT Device - Electronic module which is accessed physically and can be controlled by IoT Node. Communication between the node and the device uses I2C, SPI or UART interface. Many devices can be handled by the same IoT Node.

Back

Node

IoT Node - Communication unit which handles one or more IoT Devices. If configured the node connects to the IoT Server and sends information about occurred events and receives commands to be executed from IoT Devices.

Back

Property

Property - An attribute, quality, or characteristic of IoT Device. Properties can be numeric or textual as well as read-only or read-write.

For example, MOD-EMTR device has the VoltageRMS property, which represents the momentary measured RMS Voltage in deci-volts and this is a read-only property.

Back

Server

OlimexIoT Server - Server to which IoT Nodes (if configured) send information about events that occurred. All reported events are logged into the server's database. The server can send to the node commands to be executed.

The server is designed to support multiple users and multiple nodes per user.

You can build your own OlimexIoT Server or use our free OlimexIoT Service.

Back

Trigger

Trigger - A condition, that causes Action to take place, when met. Conditions can be event-based (execute when temperature passes over 25°C) or time-based (execute action every monday).
